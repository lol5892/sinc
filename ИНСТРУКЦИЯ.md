# Очень подробная инструкция (с нуля)

Здесь расписано **всё по шагам**. В конце есть блок: **что можно не делать** сразу.

---

## Что у тебя в итоге получится

1. **Бот в Telegram** — пишешь ему команды, он отвечает.
2. **План недели** — открывается как мини-приложение (красивая сетка с временем). Для этого нужен **интернет-сайт по https** (облако) — это отдельный большой шаг, его **можно отложить**.
3. **Напоминания** — когда настроишь облако и время в деле, бот пришлёт сообщение вам двоим.

Пока без облака ты всё равно можешь: **узнать свои id**, **потрогать план в браузере на компьютере**.

---

## Что должно быть у тебя перед началом

| Вещь | Зачем |
|------|--------|
| Компьютер с Windows | Где запускается программа |
| Интернет | Скачать программы, Telegram общается с сервером |
| Установленный **Cursor** (или VS Code) | Открыть папку проекта и терминал |
| Установленный **Node.js** (LTS) | Чтобы работала команда `npm`. Скачать: https://nodejs.org — кнопка **LTS**, установить «далее-далее» по умолчанию. |
| Два телефона с Telegram | Ты и жена (позже) |

**Проверка Node.js:** нажми `Win + R`, введи `cmd`, Enter. В чёрном окне напиши:

```text
node -v
```

Если видишь что-то вроде `v22.x` или `v20.x` — ок. Если «не является внутренней командой» — Node не установлен, поставь с сайта выше.

## Шаг 1. Открыть папку проекта

1. Папка проекта называется **`sinc`** (у тебя она на Рабочем столе в OneDrive).
2. В **Cursor**: меню **File → Open Folder** (Файл → Открыть папку) → выбери папку **`sinc`**.
3. Слева в списке файлов должны быть `package.json`, папка `src`, файл `.env` или `.env.example`.

---

## Шаг 2. Один раз установить зависимости проекта

1. В Cursor открой **терминал**: меню **Terminal → New Terminal** (Терминал → Новый терминал). Внизу появится панель с текстом.
2. Убедись, что в заголовке терминала или в строке пути фигурирует **`sinc`** (корень проекта). Если нет — в терминале выполни (подставь свой путь, если другой):

```text
cd "C:\Users\lol58\OneDrive\Рабочий стол\sinc"
```

3. Выполни команду (скопируй целиком, Enter):

```text
npm install
```

4. Подожди 1–3 минуты. В конце не должно быть красного **ERROR**. Жёлтые **warn** можно не трогать.

Этот шаг **один раз** (или после сильных изменений в проекте).

---

## Шаг 3. Создать бота и получить токен (в Telegram)

1. Открой Telegram на телефоне или компьютере.
2. В поиске найди **@BotFather** (с синей галочкой).
3. Нажми **Start** / **Запустить**.
4. Отправь ему сообщение (скопируй):

```text
/newbot
```

5. Он спросит **имя бота** — любое человеческое, например: `Наш план недели`.
6. Потом спросит **username** — латиницей, в конце **bot**, например: `ivan_week_plan_bot`. Если занято — попробуй другое.
7. В ответ BotFather пришлёт **длинный токен** — строка вида `7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxx`.  
   **Это секрет.** Не кидай в чаты, не публикуй в интернете.

---

## Шаг 4. Положить токен в файл `.env`

1. В папке `sinc` найди файл **`.env`**. Если его нет — скопируй файл **`.env.example`**, вставь копию, переименуй копию в **`.env`**.
2. Открой **`.env`** в Cursor (двойной щелчок).
3. Найди строку:

```text
BOT_TOKEN=
```

4. Сразу после знака `=` **без пробела** вставь **весь** токен от BotFather. Пример (твой токен будет другой):

```text
BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

5. Сохрани файл: **Ctrl+S**.

---

## Шаг 5. Запустить программу на компьютере

1. Терминал снова в корне папки `sinc` (см. шаг 2).
2. Выполни:

```text
npm run dev
```

3. Подожди 10–20 секунд. Должно быть примерно так:
   - что-то про **Vite** и порт **5173**;
   - строка про **API** и порт **3001**;
   - строка **«Бот на связи…»** или про long polling.

4. **Окно терминала не закрывай**, пока хочешь пользоваться ботом с этого ПК. Закрыл терминал или нажал Ctrl+C — бот перестаёт отвечать, пока снова не запустишь `npm run dev`.

---

## Шаг 6. Узнать свой Telegram ID (команда `/myid`)

1. В Telegram найди **своего** бота по тому **username**, который ты задал в BotFather (не @BotFather!).
2. Нажми **Start** / **Запустить** (если просит).
3. Отправь боту **точно** (маленькими буквами, слэш слева):

```text
/myid
```

4. Бот должен ответить **числом** — это твой ID. Скопируй его.

5. Пусть **жена** откроет **того же** бота и тоже отправит `/myid`. Скопируй **второе** число.

Если **тишина** — см. раздел «Если бот не отвечает» внизу.

---

## Шаг 7. Вписать оба ID в `.env`

1. Открой снова файл **`.env`**.
2. Найди строку:

```text
TELEGRAM_ALLOWED_IDS=...
```

3. Замени на **два числа через запятую, без пробелов**: сначала твой id, потом жены. Пример:

```text
TELEGRAM_ALLOWED_IDS=123456789,987654321
```

4. Сохрани **Ctrl+S**.
5. В терминале останови программу: **Ctrl+C** (можно два раза).
6. Снова запусти:

```text
npm run dev
```

Теперь бот знает, что это «ваша семья».

---

## Шаг 8. Открыть план в браузере на компьютере (без телефона в Telegram)

Мини-приложение внутри Telegram требует **https**. На компьютере проще — обычный браузер.

1. В **`.env`** добавь **новую строку** (подставь **своё** число из шага 6):

```text
VITE_DEV_USER_ID=твой_id_цифрами
```

2. Сохрани `.env`, перезапусти `npm run dev` (Ctrl+C, потом снова команда).
3. Открой браузер (Chrome и т.д.).
4. В адресной строке набери:

```text
http://localhost:5173
```

5. Должен открыться **план недели**. Можно добавлять дела, тянуть блоки.

**Жена с своего компа** так же не откроет твой `localhost` — это только твой ПК. Общий доступ через интернет = облако (ниже).

---

## Шаг 9. Команда `/start` у бота

Отправь боту:

```text
/start
```

- Если **`WEB_APP_URL`** в `.env` **пустой**, бот напишет, что нужен https-адрес — **это нормально**, план в браузере на шаге 8 всё равно работает.
- Если позже заполнишь `WEB_APP_URL`, появится кнопка открыть мини-приложение в Telegram.

---

# Что **не обязательно** делать сразу

| Вещь | Когда нужна |
|------|-------------|
| **`WEB_APP_URL`** | Только когда хочешь открывать план **из Telegram на телефоне** как приложение. Нужен сайт в интернете с **https**. |
| **Облако (Railway, Render и т.д.)** | Тоже для телефона и Telegram. На ПК в браузере можно без этого. |
| **`WEBHOOK_BASE_URL`** в `.env` | Вообще можно **не трогать** и **не добавлять**, пока ты не разбираешься. Для дома с `npm run dev` нужен только **пустой** вебхук (так и задумано). |
| **Жена сразу** | Можно сначала настроить только себя, второй id добавить потом. |
| **`npm run build`** | Нужно перед выкладкой на сервер в интернете, не для обычной проверки дома. |

# Если бот не отвечает на `/myid`

Проверь по порядку:

1. **Запущен** ли `npm run dev` и терминал **не закрыт**?
2. Пишешь **своему** боту, а не @BotFather?
3. В `.env` **нет** строки `WEBHOOK_BASE_URL=...` с чужим адресом? Если есть — **удали** её, сохрани `.env`, перезапусти `npm run dev`.
4. В терминале есть **красный текст**? Скопируй его и спроси в чате с ИИ.

---

# Кратко: минимум для «пощупать у себя»

1. Node.js установлен.  
2. В папке `sinc`: `npm install` (один раз).  
3. Токен в `.env` → `BOT_TOKEN=...`  
4. `npm run dev` — не закрывать.  
5. Своему боту: `/myid` → два id в `TELEGRAM_ALLOWED_IDS` → перезапуск `npm run dev`.  
6. В `.env`: `VITE_DEV_USER_ID=твой_id` → перезапуск → браузер `http://localhost:5173`.

Остальное — когда захочешь **телефон + Telegram** и готов разок зарегистрироваться на хостинге.
